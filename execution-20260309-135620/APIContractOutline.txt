API Contract Outline

1. GET /api/youth-accounts/{youthAccountId}/dashboard
- Description: Retrieve youth account dashboard (balance, limit, recent transactions)
- Request: Path param youthAccountId
- Response:
  {
    "youthAccountId": "string",
    "accountBalance": "decimal",
    "spendingLimit": "decimal",
    "currency": "string",
    "recentTransactions": [
      {
        "transactionId": "string",
        "date": "datetime",
        "description": "string",
        "amount": "decimal"
      }
    ]
  }
- Error Handling:
  - 403 Unauthorized (ownership/OAuth failure)
  - 500 Internal Error
- Performance: <500ms

2. POST /api/youth-accounts/{youthAccountId}/fund-transfer
- Description: Transfer funds from parent to youth account
- Request:
  {
    "sourceAccountId": "string",
    "transferAmount": "decimal",
    "currency": "string"
  }
- Response:
  {
    "transferId": "string",
    "status": "SUCCESS",
    "updatedBalance": "decimal"
  }
- Error Handling:
  - 400 Insufficient funds
  - 400 Invalid account mapping
  - 502 Transfer service failure

3. PUT /api/youth-accounts/{youthAccountId}/spending-limit
- Description: Set weekly spending limit
- Request:
  {
    "weeklyLimit": "decimal"
  }
- Response:
  {
    "youthAccountId": "string",
    "weeklyLimit": "decimal",
    "status": "UPDATED"
  }
- Error Handling:
  - 400 Validation error (invalid values)
  - 403 Unauthorized

4. GET /api/youth-accounts/{youthAccountId}/transactions
- Description: Retrieve youth account transaction history
- Request Query Params:
  - startDate
  - endDate
  - page
  - pageSize
- Response:
  {
    "transactions": [
      {
        "transactionId": "string",
        "date": "datetime",
        "merchant": "string",
        "amount": "decimal",
        "transactionType": "debit | credit"
      }
    ],
    "pagination": {
      "page": "number",
      "pageSize": "number",
      "totalRecords": "number"
    }
  }
- Error Handling:
  - 403 Unauthorized
  - 500 Internal Error

Authentication: OAuth2 Bearer Token
Authorization: Parent must own youth account (RBAC)
Audit Logging: All fund transfers and spending limit changes
Input Validation: Required fields, valid values
Output Filtering: Sensitive data omitted
OpenAPI: To be defined in SCIB-30 (Swagger)
